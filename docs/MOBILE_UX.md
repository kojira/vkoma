# vKoma モバイルUX設計書

> **Version:** 1.0.0  
> **作成日:** 2026-03-18  
> **対象:** スマホUI全面再設計（Phase 1 MVP）

---

## 1. 現状の問題点

### 1.1 下タブバーの常時表示
- タイムライン/アセット/パラメータの3タブが常に画面下部を占有
- 最も頻繁に使うAIターミナルが「タイムライン」タブ内に埋め込まれている
- アセット/パラメータは使用頻度が低いのに常時スペースを消費

### 1.2 コントロールボタンの横スクロール
- 14個のボタンが1行の `overflow-x-auto` で配置
- 右端のボタン（Y, N等）が見えず、スクロールが必要
- タッチターゲットが小さい（min-h-[28px]、推奨は44px）

### 1.3 設定ボタンの重なり
- `fixed bottom-20 right-4` の⚙ボタンがタブバーと視覚的に重なる
- タップ領域が干渉する可能性

### 1.4 ターミナル表示エリアの狭さ
- プレビュー + タイムライン + ChatPanel + タブバー + 設定ボタンが全て1画面に
- ターミナルの実質表示領域が画面の30%程度

---

## 2. 設計原則

### 2.1 優先順位（操作頻度順）
1. **AIターミナル** — 最頻使用。画面の最大面積を占めるべき
2. **プレビュー** — 確認用。常時表示だが小さくてOK
3. **タイムライン** — たまに確認。必要時のみアクセス
4. **アセット** — BGM追加時のみ。隠しておいてOK
5. **パラメータ** — 微調整時のみ。隠しておいてOK

### 2.2 デザイン方針
- **ターミナルファースト**: デフォルト画面はターミナル最大化
- **オンデマンドアクセス**: 低頻度機能はボトムシートで呼び出し
- **片手操作**: 重要な操作は画面下半分に配置
- **cli-chat-poc踏襲**: 実績あるコントロールボタンの2行グリッドレイアウト

## 4. コンポーネント設計

### 4.1 コントロールボタン（cli-chat-poc方式に統一）

**現状（問題）:**
```
[⇧][Ctrl][Enter][Tab][Esc][Ctrl+C][PgUp][←][↑][↓][→][PgDn][Y][N]
                                    ← 横スクロール必要 →
```

**改善（2行グリッド）:**
```
行1: [⇧] [Ctrl] [Enter] [Tab] [Esc] [Ctrl+C]     ← 6列グリッド
行2: [PgUp] [←] [↑] [↓] [→] [PgDn] [Y] [N]      ← 8列グリッド
```

- `display: grid` で均等配置（横スクロール不要）
- `min-height: 44px` のタッチターゲット（iOS HIG準拠）
- `touch-action: manipulation` でダブルタップズーム防止
- `gap: 8px` で適度な間隔

### 4.2 ツールバー（タブバー置き換え）

従来の3タブ常時表示を廃止。代わりに**コンパクトなツールバー**を配置。

```
┌─────────────────────────────────────────┐
│ [🎬] [🎵] [⚙] [New Chat]              │
└─────────────────────────────────────────┘
```

- 各ボタンはボトムシートを開くトグル
- `New Chat` ボタンもここに統合（ヘッダーから移動）
- 1行に収まるコンパクトなデザイン
- アイコン + 短いラベル

### 4.3 ボトムシート

- **ドラッグハンドル**: 上部に灰色のバー（5px × 40px）
- **スナップポイント**: 0%（閉じ）/ 50% / 70%
- **背景タップで閉じる**: シートの外側をタップで閉じる
- **スワイプダウンで閉じる**: 下方向スワイプで閉じる
- アニメーション: `transition: transform 300ms ease-out`

### 4.4 ヘッダー（モバイル最適化）

```
┌─────────────────────────────────────────┐
│ [←] ProjectName            [⚙] [📹]   │
└─────────────────────────────────────────┘
```

- 1行に収める（現状のflex-wrapを廃止）
- プロジェクト名は `truncate` で省略
- 保存ボタンは廃止（自動保存前提）/ または⚙内に移動
- Export/DownloadはSettingsModal内に移動

## 5. レイアウト構造（モバイル）

```
<div class="mobile-layout">           ← flex-col, h-screen
  <Header />                          ← flex-shrink-0, コンパクト
  <PreviewCanvas />                   ← flex-shrink-0, 小さめ固定高
  <div class="terminal-area">         ← flex-1, min-h-0（残り全部）
    <ChatPanel />
  </div>
  <div class="ime-bar" />             ← flex-shrink-0
  <div class="control-keys" />        ← flex-shrink-0, 2行グリッド
  <div class="toolbar" />             ← flex-shrink-0, ツールバー
  <BottomSheet />                     ← fixed, z-50
</div>
```

### レイアウト比率（375×667 iPhone SE想定）

| 要素 | 高さ | 割合 |
|---|---|---|
| ヘッダー | 48px | 7% |
| プレビュー | ~120px (16:9比率で) | 18% |
| ターミナル | ~230px | 35% |
| IME入力バー | 48px | 7% |
| コントロールキー | 104px (44px×2 + gap) | 16% |
| ツールバー | 48px | 7% |
| safe-area | ~env() | varies |

→ ターミナルが画面の約35%を占有（現状の~15%から大幅改善）

### プレビュー非表示時

ツールバーにプレビューの表示/非表示トグルを追加。非表示にするとターミナルが~53%に拡大。

## 6. デスクトップ（PC）レイアウト維持

PCでは既存レイアウトを維持。変更はモバイル（`md:` 未満）のみ。

```
PC: サイドバー（ParamPanel + AssetLibrary）| メイン（Preview + Timeline + ChatPanel内蔵） | ChatPanel
モバイル: 縦スタック（Header → Preview → Terminal → Controls → Toolbar）
```

ブレークポイント: `md` (768px) を境界とする。

## 7. 実装方針

### 7.1 App.tsx の変更

1. モバイル用の状態管理を追加:
   - `bottomSheetContent: "timeline" | "assets" | "params" | null`
   - `previewExpanded: boolean`
   - `previewVisible: boolean`
2. 従来の下タブバー（`<nav>` + 3ボタン）を削除
3. 固定設定ボタン（`fixed bottom-20`）を削除
4. モバイルレイアウトを再構築:
   - ヘッダー → プレビュー → ターミナル → IME → コントロールキー → ツールバー
5. ボトムシートコンポーネントを追加
6. プレビュー拡大モードを追加

### 7.2 ChatPanel.tsx の変更

1. コントロールキーのレイアウトをcli-chat-poc方式に変更:
   - `overflow-x-auto` の1行 → 2行 `grid` レイアウト
   - 行1: 6列（⇧, Ctrl, Enter, Tab, Esc, Ctrl+C）
   - 行2: 8列（PgUp, ←, ↑, ↓, →, PgDn, Y, N）
2. ボタンサイズを `min-h-[44px]` に拡大
3. ボタンのborder-radiusを `rounded-xl` (12px) に変更
4. `pb-12` を削除（タブバーがなくなるため不要）
5. フォントサイズを `text-sm` (14px) に拡大

### 7.3 新規コンポーネント

- `BottomSheet.tsx`: 汎用ボトムシート（ドラッグハンドル + スワイプ + 背景タップ閉じ）
- `MobileToolbar.tsx`: ツールバー（タイムライン/アセット/パラメータ/New Chatボタン）

## 8. インタラクション詳細

### 8.1 ボトムシートの開閉
- **開く**: ツールバーのボタンをタップ
- **閉じる**: 背景タップ / 下スワイプ / 閉じるボタン
- **同じボタン再タップ**: 閉じる（トグル動作）
- **別のボタンタップ**: コンテンツ切り替え

### 8.2 プレビュー拡大
- **拡大**: プレビューをタップ
- **縮小**: ✕ボタン / 背景タップ
- **非表示**: ツールバーのプレビュートグル

### 8.3 キーボード表示時
- IME入力にフォーカス → ソフトキーボード表示
- `visualViewport` APIでビューポートの高さを追従
- コントロールキーとツールバーは非表示（キーボードが代替）

---

## 9. アクセシビリティ

- 全タッチターゲット: 最小44×44px
- `aria-label` を全ボタンに設定
- ボトムシート: `role="dialog"` + `aria-modal="true"`
- フォーカストラップ: ボトムシート表示中は背景要素にフォーカスを移さない
- `prefers-reduced-motion` 対応: アニメーションを無効化するオプション
